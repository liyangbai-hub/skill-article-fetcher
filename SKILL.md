# article-fetcher

一个链接 → 三选一处理。脚本只做"转换"这类机械活（提取内容 / 生成 PDF），**「提炼摘要+语义标签」由你这个调用方智能体来做**——这是刻意的分工：先用脚本把文章转成 Markdown+本地图片（零模型 token），再由你结合图文理解，省 token 且可复用。

`<SKILL_DIR>` = 本 SKILL.md 所在目录。调用脚本前先解析它。

## 安装（首次）

```bash
cd <SKILL_DIR>/scripts && npm install
```

`npm install` 会尝试装无头浏览器内核（`playwright install chromium`，约 150MB）。**只做"提取内容"且多为静态/微信链接时，即使没装 chromium 也能用**（走纯 HTTP 快路径）；"生成 PDF"和 JS 渲染站的兜底才需要 chromium。

## 三种处理方式

### ① 提取内容（content）

```bash
# 默认：正文回发对话框；同时把 md + 本地图片落地到 ./article-output/<标题>/
node <SKILL_DIR>/scripts/fetch-article.mjs --mode content "<url>"

# 指定目录：落地到 <dir> 而非默认目录
node <SKILL_DIR>/scripts/fetch-article.mjs --mode content --out <dir> "<url>"
```

- **基础原则：全部文字 + 全部图片都要拿到。** 两种情况都会**下载本地图片**——不带 `--out` 默认存到当前目录 `./article-output/`，带 `--out` 存到指定目录。
- 每篇一个独立文件夹 `<日期>_<标题>/`，内含 `<日期>_<标题>.md` + `images/`：正文图片重写为本地相对路径并打 `图NNN` 标识（扩展名按真实 Content-Type 判定），文末附「图片清单」表（编号↔本地文件↔原始链接）。日期取文章发布时间，取不到用抓取当天。同名产物已存在时自动追加 `_2`、`_3`，不会静默覆盖。
- 不带 `--out` 时，脚本**额外**把 Markdown 正文打到 **stdout**，你可直接转述/回发用户；落地的本地文件位置见 stderr 日志。

### ② 生成 PDF（pdf）

```bash
node <SKILL_DIR>/scripts/fetch-article.mjs --mode pdf --out <dir> "<url>"
```

PDF 必须落盘，输出 `<dir>/<日期>_<标题>/<日期>_<标题>.pdf`，脚本回打路径。未给 `--out` 时默认存到当前目录下 `./article-output/`。PDF 走无头浏览器渲染实页，排版接近浏览效果。

### ③ 提炼摘要 + 语义标签（由你来做）

**脚本不做摘要**。你按下面流程做（先转换、再理解，结合图片）：

**第 1 步 · 转换（零模型 token）** — 把素材落到一个**临时工作目录**（不要污染用户目录）：

```bash
node <SKILL_DIR>/scripts/fetch-article.mjs --mode content --out /tmp/af-<时间戳> "<url>"
```

拿到 `<日期>_<标题>.md`（正文，图片以 `图NNN` 标识，文末有图片清单）和 `images/` 本地图片。用 `--out` 时脚本 stdout 只回路径、不回正文，正好供你内部读取。

**第 2 步 · 图文理解（必须结合图片，不能只读文字）** — 这是硬要求：

- 读正文文字的同时，**必须打开 `images/` 里的图片逐张用图像理解**，把图里的信息（表格数据、图表趋势、流程结构、示意关系）纳入摘要。**只看文字、不看图，等于没做完。**
- prompt 简短精确，按图片类型定向提取（如"提取这张表的所有数据与关键数字"）。装饰图（页眉尾、二维码、纯海报）可跳过；脚本已剔除分隔线/占位小图，剩下的基本都是内容图。
- 文字与图片信息要交叉印证——但这是**你内部的核验动作，不是摘要的内容**。印证结果怎么进摘要，见下面「摘要写法约束」。
- 脚本已尽量过滤推广行（扫码/报名/课程/优惠等），残留的推广内容继续忽略。

#### 摘要写法约束（硬要求：图片是输入，不是话题）

看图是为了**把图里的信息变成摘要的论据**，不是为了在摘要里汇报"我看了图"。违反以下任何一条都算没做好：

- **以文章的论点/事实组织摘要，不要以图片组织摘要。** 把图中的数据、趋势、结论直接当成内容写进对应论点（如"54 位经济学家预测中值 4.2%、最高 4.3%"），就像这些信息本来就写在正文里一样。
- **不要按图叙述**：禁止"图 2 显示…""（图 5）""从图 11 可以看到…"这类以图为主语/括注的行文。读者不持有图片，图号对他没有意义。确需指明信息来源时，说图的出处而不是图号（如"美银统计表显示…""高盛细项预测…"）。
- **不要设"图文一致性"专节，不要逐条汇报一致性。** 交叉印证的默认结果（一致）保持沉默；**只有发现实质矛盾、明显口径差异或正文误读图表时**，才在摘要末尾用一两句话点出（如"正文称跌 4%/7%，原表平均值实为 -3.5%/-6.6%，正文做了约整"）。没有矛盾就一个字都不要提。
- **不要描述工作过程**：禁止"已完成全文与 N 张内容图的交叉阅读""结合图片理解后"之类的过程性开场白。直接给标签抬头和摘要。

**第 3 步 · 识别标签** — 从正文（结合图片要点）识别关键词与主题，生成一组语义标签（如 `#行业研究 #AI #成本结构`）。

**第 4 步 · 产出**

⚠️ **只要求摘要时，不要把原文全文 / 图片清单 / 抓取路径发给用户——只给"标签抬头 + 摘要"。** 原文抓取只是你内部理解的中间步骤。

- **回发对话框**：开头是**醒目的标签抬头**，再接摘要正文。格式：

  ```
  🏷 标签：#标签1 #标签2 #标签3
  📄 <标题>｜🗓 <日期>｜🔗 <来源域名>

  <摘要正文：结论先行，再展开关键依据/数据（图中信息直接融入论据，不按图叙述）；仅当存在实质矛盾或不确定项时，末尾一两句点出>
  ```

- **要求"生成文件"时**：把"标签抬头 + 摘要"写成文件；文件名要带**日期 + 标题 + 标签**，例如
  `2026-06-04_如何用Codex了解陌生行业_AI_行业研究_效率工具.md`。
- **批量 + 要求生成文件**：**每篇各自一个独立文本文件**，各自用自己的「日期_标题_标签」命名，放到用户指定目录（未指定就放一个清晰的汇总目录）。不要把多篇塞进一个文件。

#### 解读详略三档（精简 / 普通 / 详细）

用户可指定解读详略；**不指定默认「普通」**。触发词如"精简解读/简要总结"、"详细解读/深度解读/详尽分析"。

> ⚠️ **关键：三档的全文+图片理解深度完全一样**（都要把全文读透、把每张内容图看懂）。**区别只在最终返回给用户的摘要篇幅**，不在理解深度——不要因为"精简"就少看图少读文。

| 档位 | 返回篇幅 | 大致形态 |
| --- | --- | --- |
| 精简解读 | 很短 | 标签抬头 + 3–5 句话讲清核心结论 |
| 普通解读（默认）| 适中 | 标签抬头 + 一两段：核心结论 + 关键依据/数据（含图中提取的数据） |
| 详细解读 | 特别详尽 | 标签抬头 + 分点展开：背景、逐项要点、图表数据、对比、结论、不确定项 |

> 第 2–4 步等价于你已有的 `wechat-article-multimodal`「图文联合分析」流程——可复用其输出模板、token 控制与失败降级规则。处理完可删除第 1 步的临时目录。

## 智能路由（脚本如何"该快则快、该兜底则兜底"）

| 情况 | 走法 |
| --- | --- |
| 微信公众号、多数静态站（content 模式）| **fast**：纯 HTTP 抓静态 HTML，秒级，不开浏览器 |
| fast **没命中任何正文容器**（疑似 JS 空壳）| **自动回退 stealth**：无头浏览器渲染后再提取 |
| 已知 JS-only 站（知乎专栏/掘金）| **直接 stealth** |
| 生成 PDF | **直接 stealth**（必须渲染实页） |

微信文章的正文和图片 `data-src` 通常就在静态 HTML 里，所以 content 模式抓微信走 fast 即可，无需每次都开浏览器。脚本 stderr 会打印每条链接实际用的策略。

**关于"短文会不会被误判回退"**：不会。判定依据是"有没有命中真正的正文容器"，而不是字数——只要命中 `#js_content` 等容器就信任（哪怕正文很短），只有连容器都没命中（疑似 JS 空壳）才回退浏览器。即便回退，内容也不会丢，只是慢一点。

**脚本内置两类清理**（提取内容/PDF 都生效）：① 文本层过滤明显的推广行（扫码/报名/课程/优惠/添加微信/阅读原文等 CTA 短行，保守只删短行不动长正文）；② 图片层剔除分隔线/占位图（任一边 ≤8px），但**绝不动真实内容图**（解析不出尺寸时一律保留）。图片按保留顺序连续编号 `图001、图002…`。

## 批量链接

一次多个链接：命令行多个、`--file urls.txt`（每行一个）、或 stdin 管道。脚本内部并发处理（`--concurrency` 默认 2；微信大批量建议 2–3 避免频控）。

对应"提炼摘要"的批量分级（你来编排，参考 wechat-article-multimodal）：

| 链接数 | 处理方式 |
| --- | --- |
| 1–2 | 顺序精读，每篇完整走完图文理解 |
| 3–5 | 顺序处理，每篇完成后提炼要点 |
| 6–15 | 分派子代理（每组 2–3 条）|
| 16+ | 子代理 + 多波执行 |

## 输出约定（按处理方式区分）

- **提取内容**：默认把正文 Markdown 回发对话框（content 不带 `--out` 时 stdout 给正文，多篇用 `===== <url> =====` 分隔），同时本地图片+md 已落到 `./article-output/<日期>_<标题>/`；用户说"存到 X"就 `--out X`。
- **摘要归纳**：**只回"标签抬头 + 摘要"，不要回原文/图片/路径**；要求生成文件时按 `<日期>_<标题>_<标签>` 给每篇单独成文（见上"提炼摘要"第 4 步）。
- **生成 PDF**：落盘 `<日期>_<标题>/<日期>_<标题>.pdf`，回路径。
- **机器可读**：加 `--json`，stdout 输出 `{ mode, results:[{url, ok, strategy, title, mdPath/pdfPath, imageDir, error}] }`，便于你解析每条结果路径。
- 进度与策略日志一律走 **stderr**，不污染 stdout。

## 失败规则

- 同一链接失败一次就别在同一方式上反复重试（脚本内部已做 fast→stealth 兜底）。content 模式 fast 失败会自动回退浏览器；若仍失败，告知用户"无法从该链接提取内容"。
- 脚本会识别反爬验证页/已删除文章（"环境异常""该内容已被发布者删除"等）并报错，不会把拦截页当正文输出——遇到这类错误直接如实告知用户，不要重试。
- 页面抓取 30s、单图 20s 超时；图片下载失败时 Markdown 保留原始远程链接，不阻断全文。
- 提炼摘要时若图片全部下载失败：先给文字结论，再基于图注/alt/邻近段落给"低置信度图像推断"并显式标注，不要把推断写成直接识别。

## 参数速查

```
--mode  content（默认）| pdf
--out   输出目录（content 指定才落地 md+本地图片；pdf 必落盘）
--file  从文件读链接（每行一个）
--concurrency  并发数（默认 2）
--json  机器可读输出
--help  帮助
```

## 依赖

- `turndown`、`cheerio`：fast 路径只需这两个（无需 chromium）。
- `playwright`（optional）：stealth 兜底与生成 PDF 时按需懒加载。

> 本技能与桌面程序 `baidu-downloader` 的「文章链接抓取」面板同源：桌面端是 `src/main/article-fetcher/engine.ts`（含队列/持久化），本技能是其不依赖 Electron 的独立 Node 版本，fast→stealth/提取/图片/Markdown 逻辑一致。详见上层 `PROJECT-GUIDE.md`。
